# 테스트전략_케이스목록

## 1. 목적
이 문서는 설비환경시뮬레이터 MVP의 테스트 범위와 우선순위를 정의한다.

## 2. 테스트 피라미드
### 2.1 단위 테스트
- 온도 계산 규칙
- 기준 온도 복귀 규칙
- 제어 메시지 파싱
- 발행 payload 최소 계약

### 2.2 통합 테스트
- MQTT 브로커 연결
- control 토픽 구독
- status/sensor 토픽 발행
- control 수신 직후 actuator status 즉시 발행
- 10 tick 누적 후 온도 변화 반영 여부
- 발행 payload wire contract 검증

## 2.3 브로커 검증 전략
- 로컬 테스트 케이스는 메시지 완결성과 코드 로직 검증의 주 수단으로 사용한다.
- 실제 연동 검증은 로컬 브로커보다 외부 테스트 브로커 기반을 우선한다.
- 외부 브로커 테스트에서 발생하는 네트워크 지연, 연결 실패, 브로커 응답 이상도 통합 테스트의 일부로 간주한다.
- 단, 외부 브로커 장애는 곧바로 코드 결함으로 단정하지 않고 연동 환경 이슈와 구현 이슈를 분리해 해석한다.
- public broker 사용 시 topic collision 방지를 위해 고유 uniq user id를 사용한다.

## 3. 1차 필수 테스트 케이스
### 3.1 단위 테스트
- 에어컨만 ON이면 10초마다 `-0.3`
- 에어컨만 켜져 있으면 온도는 `22도` 이하로 내려가지 않는다
- 컨베이어만 ON이면 10초마다 `+0.2`
- 컨베이어 과열 모드 ON이면 기존 가열량에 `+1.0`이 추가된다
- 컨베이어만 켜져 있으면 온도는 `50도` 이상으로 올라가지 않는다
- 에어컨과 컨베이어가 동시에 ON이면 10초마다 `-0.1`
- 에어컨과 컨베이어 과열 모드가 동시에 ON이면 10초마다 `+0.9` 상승한다
- 모든 장치가 OFF면 기준 온도 `25도`로 `0.1`씩 복귀
- 기준 온도 복귀 시 overshoot가 발생하지 않는다
- 컨베이어와 에어컨 제어는 `power=on/off`만 허용하고 그 외 값은 거부한다
- 잘못된 JSON 또는 `power` 누락 payload는 `null` 처리한다
- `reason`이 있으면 string이어야 하며 그 외 타입은 거부한다
- `MQTT_UNIQ_USER_ID` 누락 시 fail-fast 한다

### 3.2 통합 테스트
- 기본 10초 정기 발행 주기마다 `temperature`, `vibration`, `conveyor-belt/status`, `aircon/status`가 발행된다
- `conveyor-belt/control`, `aircon/control` 수신 시 해당 status가 즉시 추가 발행된다
- `conveyor-belt/control`의 `overheatMode` 변화가 `conveyor-belt/status`에 즉시 반영된다
- MQTT 제어 메시지 수신 후 내부 상태가 변경된다
- 에어컨/컨베이어 동시 가동 후 10 tick 경과 시 온도가 `24.9`가 된다
- 과열 모드 ON + 컨베이어 ON + 에어컨 ON 후 10 tick 경과 시 온도가 `25.9`가 된다
- 지원하지 않는 power alias는 수신해도 내부 상태를 오염시키지 않는다
- `power` 누락 또는 잘못된 타입 payload는 수신해도 내부 상태를 오염시키지 않는다
- `start()` 호출 시 tick scheduler가 자동 발행을 시작하고 `stop()` 이후 발행을 멈춘다
- `status/sensor` payload는 필드 집합과 최소 타입 규약을 만족한다
- malformed raw MQTT payload를 수신해도 상태는 오염되지 않고 이후의 정상 제어 메시지를 계속 처리한다

## 4. 2차 확장 테스트 케이스
- 잘못된 payload 수신 시 상태를 오염시키지 않는다
- 브로커 재연결 이후 제어 메시지 수신을 다시 수행한다
- 장시간 tick 누적 시 발행이 누락되지 않는다
- 발행 payload 스키마가 계약과 일치한다
- 외부 테스트 브로커 기준 publish/subscribe 연동이 실제 네트워크 환경에서도 성립한다
- 외부 테스트 브로커에서도 고유 uniq user id 기반으로 독립 테스트 공간을 유지한다

## 5. 완료 판정
아래 조건이 동시에 참이어야 MVP 기능이 완료다.

- 1차 필수 테스트 케이스 전부 녹색
- `npm run verify` 통과
- 관련 설계 문서 최신 상태 유지
