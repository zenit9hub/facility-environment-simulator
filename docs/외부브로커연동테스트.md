# 외부브로커연동테스트

## 1. 목적
이 문서는 외부 테스트 브로커를 사용해 설비환경시뮬레이터의 실제 MQTT 연동을 검증하는 방법을 정의한다.

## 2. 기본 원칙
- 로컬 테스트는 메시지 완결성과 코드 로직 검증의 주 수단이다.
- 외부 브로커 테스트는 실제 네트워크 환경에서 publish/subscribe 경로가 성립하는지 확인하는 목적이다.
- 외부 브로커 장애나 네트워크 지연도 연동 테스트의 일부로 본다.
- 단, 실패 원인은 구현 문제와 환경 문제를 분리해서 해석한다.

## 3. 기본 브로커
- 기본 외부 테스트 브로커 주소: `mqtt://broker.emqx.io:1883`
- 필요 시 `EXTERNAL_BROKER_URL`로 변경할 수 있다.

## 4. uniq user id 분리
public broker에서는 고정 토픽을 그대로 사용하면 다른 사용자와 충돌할 수 있다.
따라서 외부 브로커 테스트는 고유한 `uniq-user-id`를 사용해야 한다.
이 프로젝트는 `MQTT_UNIQ_USER_ID`와 `EXTERNAL_TEST_UNIQ_USER_ID`를 통해 네임스페이스를 분리할 수 있다.

## 5. 실행 명령
### 5.1 외부 브로커 자동 테스트
```bash
npm run test:external
```

### 5.2 브로커 주소 지정
```bash
EXTERNAL_BROKER_URL=mqtt://broker.emqx.io:1883 npm run test:external
```

### 5.3 uniq user id 지정
```bash
EXTERNAL_BROKER_URL=mqtt://broker.emqx.io:1883 \
EXTERNAL_TEST_UNIQ_USER_ID=manual-run-001 \
npm run test:external
```

## 6. 현재 검증 범위
- 외부 브로커에 실제 연결
- 고유 uniq user id 사용
- 제어 메시지 publish
- 시뮬레이터 상태 반영
- 상태값/센서값 subscribe
- 수신된 센서 payload 계약 검증
- 메시지 묶음 ID 없이도 topic별 독립 payload가 정상 발행되는지 검증

## 7. 해석 기준
- `npm run verify`는 로컬 검증 게이트다.
- `npm run test:external`은 외부 연동 검증 게이트다.
- 외부 브로커 테스트 실패는 시스템 문제를 발견한 것으로 간주하되, 원인 분석 시 브로커/네트워크 상태를 함께 기록한다.
