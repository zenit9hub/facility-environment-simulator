# 실행가이드

## 1. 목적
이 문서는 설비환경시뮬레이터를 실제 MQTT 브로커에 연결해 실행하는 방법을 설명한다.

## 2. 기본 실행 전제
- MQTT 브로커가 준비되어 있어야 한다.
- `MQTT_UNIQ_USER_ID`가 필요하다.
- 기본 브로커 주소는 `mqtt://broker.emqx.io:1883`이다.
- 실제 연동 검증은 외부 테스트 브로커 사용을 우선 권장한다.

## 3. 실행 명령
### 3.1 개발 모드
```bash
npm run dev
```

TypeScript 소스를 직접 실행한다.

### 3.2 시작 명령
```bash
npm start
```

빌드 후 실행한다.

## 4. 지원 환경변수
- `MQTT_UNIQ_USER_ID`
- `MQTT_BROKER_URL`

## 5. 예시
```bash
MQTT_UNIQ_USER_ID=student-01 npm run dev
```

```bash
MQTT_UNIQ_USER_ID=student-01 MQTT_BROKER_URL=mqtt://broker.emqx.io:1883 npm run dev
```

## 6. 기본 동작
- 실행 시 브로커에 연결한다.
- 제어 토픽을 구독한다.
- 내부 scheduler는 기본 1초 tick으로 동작한다.
- 상태/센서 메시지는 기본 10초마다 정기 발행한다.
- actuator control을 받으면 해당 status를 즉시 추가 발행한다.
- `SIGINT`, `SIGTERM` 수신 시 안전하게 종료한다.

## 7. 종료 방법
- 터미널에서 `Ctrl+C`

## 8. 점검 포인트
- 브로커 연결 로그가 보이는가
- 10초 주기 정기 발행이 지속적으로 발생하는가
- control 수신 직후 해당 status가 즉시 추가 발행되는가
- 종료 시 stop 로그가 출력되는가

## 9. 외부 테스트 브로커 사용 원칙
- 로컬 테스트는 메시지 완결성과 내부 로직 검증에 충분하다고 본다.
- 실제 MQTT 연동 검증은 외부 테스트 브로커를 기준으로 수행한다.
- 외부 브로커에서 발생하는 연결 실패, 지연, publish/subscribe 이상도 연동 테스트의 일부로 본다.
- 이 경우 원인 분석은 `시뮬레이터 구현 문제`와 `브로커 또는 네트워크 환경 문제`를 구분해서 기록한다.
- public broker 사용 시에는 `MQTT_UNIQ_USER_ID`를 사용자별로 고유하게 지정해야 한다.
