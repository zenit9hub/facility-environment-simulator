# 클론_실행_가이드

이 문서는 처음 저장소를 내려받아 설비환경시뮬레이터를 실행하는 절차를 안내합니다.

## 1. 준비물 확인

아래 도구가 설치되어 있어야 합니다.

```bash
git --version
node --version
npm --version
```

Node.js는 `20.19.4` 이상을 권장합니다.

## 2. 저장소 클론

GitHub 접근 권한이 있는 계정으로 아래 저장소를 클론합니다.

```bash
git clone https://github.com/zenit9hub/facility-environment-simulator.git
```

프로젝트 폴더로 이동합니다.

```bash
cd facility-environment-simulator
```

SSH를 사용하는 경우:

```bash
git clone git@github.com:zenit9hub/facility-environment-simulator.git
```

## 3. 의존성 설치

```bash
npm install
```

## 4. 환경설정 파일 생성

`.env.example`을 복사해 `.env`를 만듭니다.

macOS/Linux:

```bash
cp .env.example .env
```

Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

## 5. 사용자 ID 설정

`.env` 파일을 열고 `MQTT_UNIQ_USER_ID`를 본인에게 할당된 값으로 변경합니다.

예:

```text
MQTT_UNIQ_USER_ID=user-01
MQTT_BROKER_URL=mqtt://broker.emqx.io:1883
CONTROL_PANEL_ENABLED=true
CONTROL_PANEL_PORT=3000
CONTROL_PANEL_AUTO_OPEN=true
```

주의:

- `MQTT_UNIQ_USER_ID`는 반드시 설정해야 합니다.
- 여러 사람이 같은 값을 사용하면 MQTT 토픽이 충돌할 수 있습니다.
- `.env` 파일은 Git에 커밋하지 않습니다.

## 6. 개발 모드 실행

```bash
npm run dev
```

정상 실행되면 콘솔에 MQTT 연결 정보와 제어 패널 URL이 출력됩니다.

기본 제어 패널:

```text
http://localhost:3000
```

브라우저가 자동으로 열리지 않으면 위 주소를 직접 엽니다.

## 7. 동작 확인

제어 패널에서 아래 항목을 확인합니다.

- 현재 온도
- 현재 진동
- 컨베이어벨트 Power
- 컨베이어벨트 Overheat Mode
- 에어컨 Power

MQTT 클라이언트 도구를 사용하는 경우 아래 토픽을 구독해 확인할 수 있습니다.

```text
kiot/{MQTT_UNIQ_USER_ID}/#
```

예:

```text
kiot/user-01/#
```

## 8. 종료

터미널에서 `Ctrl+C`를 누릅니다.

## 9. 자주 발생하는 문제

`.env` 파일이 없는 경우:

```text
node: .env: not found
```

조치:

```bash
cp .env.example .env
```

`MQTT_UNIQ_USER_ID`가 없는 경우:

```text
MQTT_UNIQ_USER_ID is required.
```

조치:

```text
MQTT_UNIQ_USER_ID=user-01
```

패널 포트가 이미 사용 중인 경우:

```text
CONTROL_PANEL_PORT=3001
```

자동 브라우저 오픈을 끄고 싶은 경우:

```text
CONTROL_PANEL_AUTO_OPEN=false
```

## 10. 검증 명령

코드와 테스트가 정상인지 확인하려면 아래 명령을 실행합니다.

```bash
npm run verify
```
