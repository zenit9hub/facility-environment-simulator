# 메타버스_3D_팩토리_연동_가이드

이 문서는 메타버스 3D 팩토리 프로젝트가 설비환경시뮬레이터의 MQTT 메시지를 사용하도록 바꾸기 위한 전달 가이드입니다.

## 1. 현재 3D 팩토리 토픽

현재 3D 팩토리는 아래 3개 토픽을 구독과 발행에 함께 사용합니다.

```text
kiot/zenit/room-a/aircon/control
kiot/zenit/room-b/aircon/control
kiot/zenit/room-c/aircon/control
```

현재 payload:

```json
{
  "power": true,
  "timestamp": "ISO 8601 시간"
}
```

## 2. 변경 방향

3D 팩토리는 더 이상 `room-a`, `room-b`, `room-c` 기준의 에어컨 제어 토픽을 사용하지 않습니다.

설비환경시뮬레이터 기준으로 아래처럼 바꿉니다.

- 구독은 `status`, `sensor` 토픽을 사용합니다.
- 발행은 `control` 토픽만 사용합니다.
- `power` 값은 boolean이 아니라 `"on"` 또는 `"off"` 문자열을 사용합니다.
- `timestamp`는 필수 payload가 아닙니다.

## 3. 3D 팩토리가 구독해야 할 토픽

`{uniq-user-id}`는 시뮬레이터의 `MQTT_UNIQ_USER_ID`와 같은 값이어야 합니다.

```text
kiot/{uniq-user-id}/factory/room-01/sensor/temperature
kiot/{uniq-user-id}/factory/room-01/sensor/vibration
kiot/{uniq-user-id}/factory/room-01/actuator/conveyor-belt/status
kiot/{uniq-user-id}/factory/room-01/actuator/aircon/status
```

예:

```text
kiot/zenit/factory/room-01/sensor/temperature
kiot/zenit/factory/room-01/sensor/vibration
kiot/zenit/factory/room-01/actuator/conveyor-belt/status
kiot/zenit/factory/room-01/actuator/aircon/status
```

## 4. 3D 팩토리가 발행해야 할 토픽

3D UI에서 장비를 수동 제어한다면 아래 control 토픽으로 발행합니다.

```text
kiot/{uniq-user-id}/factory/room-01/actuator/conveyor-belt/control
kiot/{uniq-user-id}/factory/room-01/actuator/aircon/control
```

예:

```text
kiot/zenit/factory/room-01/actuator/conveyor-belt/control
kiot/zenit/factory/room-01/actuator/aircon/control
```

## 5. Payload 매핑

### 온도 센서 수신

Topic:

```text
kiot/{uniq-user-id}/factory/room-01/sensor/temperature
```

Payload:

```json
{
  "value": 25.6
}
```

3D 반영 예:

- 온도 게이지
- 공장 위험도 색상
- 온도 텍스트 UI

### 진동 센서 수신

Topic:

```text
kiot/{uniq-user-id}/factory/room-01/sensor/vibration
```

Payload:

```json
{
  "value": 1
}
```

3D 반영 예:

- 컨베이어 동작 애니메이션
- 진동 표시 UI

### 컨베이어 상태 수신

Topic:

```text
kiot/{uniq-user-id}/factory/room-01/actuator/conveyor-belt/status
```

Payload:

```json
{
  "power": "on",
  "overheatMode": "off"
}
```

3D 반영 예:

- `power === "on"`이면 컨베이어 가동 애니메이션 ON
- `power === "off"`이면 컨베이어 가동 애니메이션 OFF
- `overheatMode === "on"`이면 과열 효과 또는 경고 색상 표시

### 에어컨 상태 수신

Topic:

```text
kiot/{uniq-user-id}/factory/room-01/actuator/aircon/status
```

Payload:

```json
{
  "power": "on"
}
```

3D 반영 예:

- `power === "on"`이면 에어컨 LED 또는 냉각 효과 ON
- `power === "off"`이면 에어컨 LED 또는 냉각 효과 OFF

## 6. 3D UI에서 제어 발행

3D UI가 장비를 직접 제어할 경우, 기존처럼 status와 같은 토픽에 다시 publish하지 말고 control 토픽으로만 publish합니다.

### 에어컨 제어

Topic:

```text
kiot/{uniq-user-id}/factory/room-01/actuator/aircon/control
```

Payload:

```json
{
  "power": "on",
  "reason": "metaverse-3d"
}
```

OFF:

```json
{
  "power": "off",
  "reason": "metaverse-3d"
}
```

### 컨베이어 제어

Topic:

```text
kiot/{uniq-user-id}/factory/room-01/actuator/conveyor-belt/control
```

Payload:

```json
{
  "power": "on",
  "reason": "metaverse-3d"
}
```

과열 모드까지 제어할 경우:

```json
{
  "power": "on",
  "overheatMode": "on",
  "reason": "metaverse-3d"
}
```

## 7. 기존 코드 변경 포인트

기존:

```js
const topics = [
  'kiot/zenit/room-a/aircon/control',
  'kiot/zenit/room-b/aircon/control',
  'kiot/zenit/room-c/aircon/control'
];
```

변경:

```js
const userId = 'zenit';
const roomId = 'room-01';
const topicRoot = `kiot/${userId}/factory/${roomId}`;

const subscribeTopics = {
  temperature: `${topicRoot}/sensor/temperature`,
  vibration: `${topicRoot}/sensor/vibration`,
  conveyorStatus: `${topicRoot}/actuator/conveyor-belt/status`,
  airconStatus: `${topicRoot}/actuator/aircon/status`
};

const publishTopics = {
  conveyorControl: `${topicRoot}/actuator/conveyor-belt/control`,
  airconControl: `${topicRoot}/actuator/aircon/control`
};
```

기존 boolean payload:

```json
{
  "power": true
}
```

변경:

```json
{
  "power": "on",
  "reason": "metaverse-3d"
}
```

boolean 변환 예:

```js
function toPower(value) {
  return value ? 'on' : 'off';
}
```

수신 처리 예:

```js
function isOn(payload) {
  return payload.power === 'on';
}
```

## 8. 구현 시 주의사항

- `control` 토픽은 명령 발행용입니다.
- `status` 토픽은 시뮬레이터가 발행하는 상태 수신용입니다.
- 3D 팩토리는 `status`를 구독해 화면을 갱신해야 합니다.
- 3D 팩토리가 버튼으로 제어할 때만 `control`로 발행합니다.
- `power`는 반드시 `"on"` 또는 `"off"` 문자열입니다.
- temperature와 vibration은 `{ "value": number }`만 사용합니다.
- `room-a`, `room-b`, `room-c` 구조는 현재 시뮬레이터와 맞지 않으므로 `factory/room-01` 기준으로 통일합니다.

## 9. 최소 완료 기준

- 3D 팩토리가 `kiot/{uniq-user-id}/factory/room-01/sensor/temperature`를 구독해 온도를 표시합니다.
- 3D 팩토리가 `kiot/{uniq-user-id}/factory/room-01/actuator/aircon/status`를 구독해 에어컨 상태를 표시합니다.
- 3D 팩토리에서 에어컨 버튼을 누르면 `aircon/control`로 `{ "power": "on" | "off" }`를 발행합니다.
- 시뮬레이터가 즉시 `aircon/status`를 발행하고, 3D 화면이 그 status를 기준으로 갱신됩니다.
